    # 解题思路

    模拟整个删除过程最直观，即构建一个长度为 nnn 的链表，各节点值为对应的顺序索引；每轮删除第 mmm 个节点，直至链表长度为 1 时结束，返回最后剩余节点的值即可。

    模拟法需要循环删除 n−1n - 1n−1 轮，每轮在链表中寻找删除节点需要 mmm 次访问操作（链表线性遍历），因此总体时间复杂度为 O(nm)O(nm)O(nm) 。题目给定的 m,nm, nm,n 取值范围如下所示，观察可知此时间复杂度是不可接受的。

    1≤n≤1051≤m≤106 1 \leq n \leq 10^5 \\ 1 \leq m \leq 10^6 1≤n≤105 1≤m≤106

    实际上，本题是著名的 “约瑟夫环” 问题，可使用 动态规划 解决。

    输入 n,mn, mn,m ，记此约瑟夫环问题为 「n,mn, mn,m 问题」 ，设解（即最后留下的数字）为 f(n)f(n)f(n) ，则有：

    • 「n,mn, mn,m 问题」：数字环为 0,1,2,...,n−10, 1, 2, ..., n - 10,1,2,...,n−1 ，解为 f(n)f(n)f(n) ；
    • 「n−1,mn-1, mn−1,m 问题」：数字环为 0,1,2,...,n−20, 1, 2, ..., n - 20,1,2,...,n−2 ，解为 f(n−1)f(n-1)f(n−1) ；
    • 以此类推……

    请注意，数字环是 首尾相接 的，为方便行文，本文使用列表形式表示。

    对于「n,mn, mn,m 问题」，首轮删除环中第 mmm 个数字后，得到一个长度为 n−1n - 1n−1 的数字环。由于有可能 m>nm > nm>n ，因此删除的数字为 (m−1)%n(m - 1) \% n(m−1)%n ，删除后的数字环从下个数字（即 m%nm \% nm%n ）开始， 设 t=m%nt = m \% nt=m%n ，可得数字环：

    t,t+1,t+2,...,0,1,...,t−3,t−2 t, t + 1, t + 2, ..., 0, 1, ..., t - 3, t - 2 t,t+1,t+2,...,0,1,...,t−3,t−2

    例如，当n=4,m=4n = 4, m = 4n=4,m=4时，第一个被删除的数字是 (4−1)%4=3(4 - 1) \% 4 = 3(4−1)%4=3，下一个计数开始的数字是4%4=04 \% 4 = 04%4=0。

    删除一轮后的数字环也变为一个「n−1,mn-1, mn−1,m 问题」，观察以下数字编号对应关系：

    「n-1, m 问题」→「n, m 问题」删除后0→t+01→t+1...→...n−2→t−2 \begin{aligned} \text{「n-1, m 问题」} && \rightarrow && \text{「n, m 问题」删除后} \\ 0 && \rightarrow && t + 0 \\ 1 && \rightarrow && t + 1 \\ ... && \rightarrow && ... \\ n - 2 && \rightarrow && t - 2 \\ \end{aligned} 「n-1, m 问题」01...n−2​​→→→→→​​「n, m 问题」删除后t+0t+1...t−2​

    设「n−1,mn-1, mn−1,m 问题」某数字为 xxx ，则可得递推关系（关键点）：

    x→(x+t)%n x \rightarrow (x + t) \% n \\ x→(x+t)%n 

    如何理解上面的公式？

    对于n−1,mn-1, mn−1,m 问题」中的任意数字xxx，我们都能推出它在「n,mn, mn,m 问题」删除一个值后的对应值为(x+t)%n(x + t) \% n(x+t)%n（应该指的是下标，本题中刚好下标等于数字本身）

    即两个问题可以对应起来，长度为n−1⇔n-1 \Leftrightarrown−1⇔ 长度为 nnn 经过第一轮删除

    换而言之，若已知「n−1,mn-1, mn−1,m 问题」的解 f(n−1)f(n - 1)f(n−1) （n−1n-1n−1问题中剩下的最后的数字），则可通过以上公式计算得到「n,mn, mn,m 问题」的解 f(n)f(n)f(n)（nnn问题中剩下的最后的数字） ，即：

    f(n)=(f(n−1)+t)%n=(f(n−1)+m%n)%n=(f(n−1)%n+m%n%n)%n(模运算的基本性质)=(f(n−1)%n+m%n)%n(m%n的结果一定小于n, 所以m%n%n=m%n)=(f(n−1)+m)%n(模运算性质的反向应用) \begin{aligned} f(n) & = (f(n - 1) + t) \% n \\ & = (f(n - 1) + m \% n) \% n \\ & = (f(n - 1) \% n + m \% n \% n) \% n \quad \text{(模运算的基本性质)}\\ & = (f(n - 1) \% n + m \% n) \% n \quad \text{(}m \% n\text{的结果一定小于n, 所以} m \% n \% n = m \% n\text{)}\\ & = (f(n - 1) + m) \% n \quad \text{(模运算性质的反向应用)} \end{aligned} f(n)​=(f(n−1)+t)%n=(f(n−1)+m%n)%n=(f(n−1)%n+m%n%n)%n(模运算的基本性质)=(f(n−1)%n+m%n)%n(m%n的结果一定小于n, 所以m%n%n=m%n)=(f(n−1)+m)%n(模运算性质的反向应用)​

    f(n)f(n)f(n) 可由 f(n−1)f(n - 1)f(n−1) 得到，f(n−1)f(n - 1)f(n−1) 可由 f(n−2)f(n - 2)f(n−2) 得到，……，f(2)f(2)f(2) 可由 f(1)f(1)f(1) 得到；

    因此，若给定 f(1)f(1)f(1) 的值，就可以递推至任意 f(n)f(n)f(n) 。而「1,m1, m1,m 问题」的解 f(1)=0f(1) = 0f(1)=0 恒成立，即无论 mmm 为何值，长度为 1 的数字环留下的是一定是数字 000 。

    以上数学推导本质是得出动态规划的 转移方程 和 初始状态 。

    # 动态规划解析：

    1. 状态定义： 设「i,mi, mi,m 问题」的解为 dp[i]dp[i]dp[i]
    2. 转移方程： 通过以下公式可从 dp[i−1]dp[i - 1]dp[i−1] 递推得到 dp[i]dp[i]dp[i]
    dp[i]=(dp[i−1]+m)%i dp[i] = (dp[i - 1] + m) \% i dp[i]=(dp[i−1]+m)%i
    1. 初始状态：「1,m1, m1,m 问题」的解恒为 000 ，即 dp[1]=0dp[1] = 0dp[1]=0
    2. 返回值： 返回「n,mn, mn,m 问题」的解 dp[n]dp[n]dp[n]

    # 复杂度分析

    • 时间复杂度 O(n)O(n)O(n) ： 状态转移循环 n−1n - 1n−1 次使用 O(n)O(n)O(n) 时间，状态转移方程计算使用 O(1)O(1)O(1) 时间；
    • 空间复杂度 O(1)O(1)O(1) ： 使用常数大小的额外空间；

    # 代码：

    function lastRemaining(n: number, m: number): number {
    const dp = [];
    dp[1] = 0; // 初始条件

    for(let i = 2; i <= n; i++) {
        dp[i] = (dp[i - 1] + m) % i;
    }

    return dp[n];
};
